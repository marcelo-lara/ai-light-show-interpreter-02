# ai-light-show-interpreter-02: Stage Virtual Canvas Interpreter

## 1. Overview
The `ai-light-show-interpreter-02` is the core "rendering engine" of the AI Light Show ecosystem. It functions as a **rasterizer and baker** that transforms mathematical intent and audio-reactive features into a production-ready, sample-accurate DMX binary stream.

By utilizing a **Stage Virtual Canvas** approach, this module moves away from traditional "fixture-based" chasers. Instead, it treats the lighting rig as a coordinate-aware 2D mesh, evaluating mathematical equations (pulses, waves, and gradients) to determine the state of every light.

## 2. Technical Specifications

### Input: Semantic Intelligence Artifacts
The interpreter consumes artifacts generated by the song analyzer:
* **Song Structure (`song_map.json`):** Used to determine section-based rendering logic (e.g., specific math functions for Verses vs. Choruses).
* **Audio Envelopes (`audio_features.json`):** Provides time-smoothed energy bands (Bass, Mid, Treble) and attenuated transients used as multipliers for equation intensity.
* **Event Timeline (`events.json`):** Triggers instantaneous mathematical "Hard Cuts" or coordinate shifts in the 2D mesh.

### The Bridge: Stage Virtual Canvas
A JSON configuration file that defines the virtual-to-physical mapping:
* **Fixture Coordinates:** Maps DMX addresses to specific $(x, y)$ coordinates on the virtual 2D grid.
* **Capability Metadata:** Defines fixture attributes (RGB, Dimmer, Pan/Tilt) so the interpreter can sample the canvas correctly.

### Output: Production-Grade DMX Binary
The final output is a `{song_id}.dmx` file following the **DMX Binary Specification**:
* **Header:** Metadata including file version, frame rate (e.g., 50 FPS), and total universe count.
* **Payload:** A flat, seekable byte-array buffer. Each frame contains raw DMX values (512 bytes per universe).
* **Performance:** Designed for high-reliability playback with zero computational overhead during the performance.

## 3. Rendering Pipeline (The Baker)

1.  **Coordinate Lookup:** For each fixture defined in the **Stage Virtual Canvas**, the system retrieves its $(x, y)$ position.
2.  **Evaluation:** For every 20ms frame (at 50 FPS), the system evaluates the current mathematical "state" of the canvas driven by the **Audio Features**.
3.  **Rasterization:** The system "samples" the color and brightness at each fixture's coordinate.
4.  **Baking:** Sampled results are written directly into the binary DMX frame.

## 4. Professional-Grade Features
* **Separation of Concerns:** Intent is stored in human-readable math/coordinates; execution is stored in lean binary.
* **Sample-Accurate Sync:** Pre-rendering eliminates real-time jitter and CPU-induced lag.
* **Volumetric Scaling:** Upgrading a rig only requires updating the **Stage Virtual Canvas** coordinates; the artistic logic remains untouched.
